PRODUCT & IMPLEMENTATION PLAN

나의 보드게임을 기록하고,
상황에 맞게 추천하고, 함께 나누는 공간

개인 보드게임 컬렉션을 빠르게 등록하고, 플레이 경험을 리뷰로 축적하며, 인원·연령·취향에 꼭 맞는 게임을 추천·공유하는 웹 서비스입니다.

Ant Design 기반 UI 한국어 우선 반응형 웹 리뷰 요청

1. 목표와 범위

첫 번째 버전(MVP)은 “내 컬렉션을 정확히 관리하고, 오늘의 인원 구성에 맞춰 고르는 것”에 집중합니다.

01 · 등록

검색으로 시작하는 빠른 입력

게임명을 입력하면 후보를 제시하고, 선택 시 인원·권장 연령·플레이 시간·난이도·장르 등 메타데이터를 초안으로 채웁니다.

02 · 정리와 리뷰

내 기준으로 다시 분류

자동 태그에 개인 태그를 더하고, 소장 상태·평점·한줄평·상세 리뷰·플레이 기록을 게임별로 관리합니다.

03 · 추천과 공유

모임에 맞는 선택

참가자 수와 연령을 입력하면 조건 충족 게임을 우선 정렬하고, 공개 가능한 게임 상세/리뷰 링크를 만듭니다.

2. 핵심 사용자 흐름

게임 이름 입력“스플랜더”처럼 한글·영문 검색
Boardlife 후보 선택검색 API의 게임 ID·연도·이미지로 본판/확장판 구분
상세 정보 자동 입력게임 상세 페이지에서 메타데이터를 읽어 폼을 채움
태그·리뷰·공유내 컬렉션과 추천 결과에 즉시 반영

3. 화면 구성

대시보드

소장 게임 수, 미플레이 수, 최근 리뷰, “오늘 4명과 할 게임” 빠른 추천 진입점.

내 컬렉션

카드/표 보기, 키워드 검색, 소장 상태·인원·난이도·태그 필터, 정렬과 일괄 태그 편집.

게임 등록

자동완성 검색 → 후보 비교 → 메타데이터 확인 폼 → 개인 태그와 소장 정보 입력.

게임 상세·리뷰

기본 정보, 내 평가, 플레이 기록, 리뷰, 자동·개인 태그, 공유용 공개 링크 제어.

모임 추천

참가자 수, 최연소/최고 연령, 원하는 시간·난이도·분위기를 입력해 추천 사유와 함께 제시.

태그 관리

자동 태그와 개인 태그를 구분해 검색·병합·숨김·새 태그 추가. 오타/중복을 관리.

4. 데이터 자동 입력 설계

단계처리결과와 예외
이름 검색서버가 /get_auto_search.php?word={검색어}를 호출합니다. 응답의 number, title, engtitle, years, photo를 후보 카드로 표시합니다.예: “splendor” 검색 시 스플렌더 본판과 대결·확장판을 연도·이미지와 함께 구분해 선택합니다.
상세 조회선택한 number로 /game/{number}를 서버에서 조회하고, 인원 범위·베스트/추천 인원·플레이 시간·권장 연령·난이도·평점·언어의존도를 추출합니다.필수 값이 없거나 형식이 바뀌면 해당 필드는 빈값으로 두고 수동 입력을 허용합니다.
정규화예: “2-4명”, “베스트: 3인”, “30-45분” 같은 화면 텍스트를 추천에 쓸 숫자 범위로 변환하고, 장르·특징은 내부 태그 사전에 매핑합니다.원문 값과 원본 URL을 함께 보관해 파싱 오류를 확인하고 추후 재처리할 수 있게 합니다.
저장Boardlife 게임 ID, 원본 URL, 마지막 조회 시각, 자동 입력값과 개인 수정값을 구분해 저장합니다.재조회 시 개인 수정값은 유지하고, 바뀐 외부 값만 비교·적용하도록 합니다.
연동 안정성 기준검색은 입력 300ms 후 실행하고 동일 검색어는 캐시합니다. 상세 조회는 게임 선택 시 1회만 호출합니다. 모든 요청은 서버에서 수행하며, 실패하면 수동 등록 폼으로 자연스럽게 전환합니다.

5. 태그 체계

자동 분류는 출발점이고, 개인 태그는 내 모임의 언어로 컬렉션을 완성합니다. 자동 태그는 원본과 연결하고 개인 태그는 언제든 생성·수정할 수 있습니다.

자동 태그 후보

전략가족파티협력추리추상 전략엔진 빌딩세트 컬렉션덱 빌딩일꾼 배치주사위협상블러핑드래프팅

개인화 태그 후보

2인 데이트아이와 함께초보 환영설명 5분생각할 거리웃긴 게임여행용재플레이 좋음주말 밤모임 필승

6. 인원·연령 기반 추천 규칙

우선순위규칙추천 화면 표시
필수 조건입력 인원이 게임의 최소~최대 인원에 포함되고, 최연소 참가자가 권장 연령 이상인 게임만 통과합니다.“4명 플레이 가능 · 10세 이상 적합”
강한 가점입력 인원이 사용자가 지정한 베스트 인원과 일치하거나, 개인 태그가 “초보 환영” 등 모임 조건과 일치합니다.“4명일 때 가장 재미있어요”
선호 가점플레이 시간, 난이도, 보유 상태, 내 평점, 최근 플레이 여부, 원하는 장르를 점수화합니다.추천 점수와 2~3개의 추천 이유
투명성추천 결과마다 제외/포함 이유를 보여주고, 필터와 정렬을 사용자가 즉시 변경할 수 있습니다.“7세 참가자 조건으로 제외됨”

7. 기술 구성 제안

영역제안이유
웹 앱Next.js + TypeScript + Ant Design대시보드·폼·테이블·태그·모달 등 관리형 UI를 일관된 컴포넌트와 토큰으로 구성합니다.
디자인 시스템Ant Design 기본 라이트 테마, Primary #1677ff, 8px 반경, 24px 기반 여백차분한 관리 도구 느낌을 유지하고 상태 색상·반응형 레이아웃을 통일합니다.
서버·DBNext.js Route Handler + PostgreSQL + Prisma개인 컬렉션, 태그, 리뷰, 플레이 기록, 공개 링크를 관계형 구조로 안전하게 저장합니다.
외부 연동Boardlife 검색 엔드포인트 + 게임 상세 페이지 서버 어댑터 + 캐시한글 게임명을 바로 검색하고, 반환된 Boardlife ID로 상세 정보까지 이어서 가져옵니다. BGG는 토큰 발급 후 보조 출처로 추가합니다.
개인 서비스·공유단일 관리자 로그인, 전체 컬렉션 읽기 전용 공개 페이지등록·수정 권한은 본인에게만 두고, 게임·리뷰·컬렉션 전체를 하나의 공개 링크로 공유합니다.

8. 구현 순서와 완료 기준

Phase 1 · 기반

프로젝트·데이터 모델·Ant Design 테마

로그인, 게임/태그/리뷰/플레이 기록 테이블, 공통 레이아웃과 네비게이션을 구성합니다.

Phase 2 · 등록

Boardlife 2단계 검색 연동과 수정 가능한 등록 폼

자동완성 검색 API → number 선택 → 상세 페이지 메타데이터 추출 → 사용자 확인·저장의 흐름을 구현합니다. 검색어 캐시, 중복 방지, 수동 등록 폴백을 포함합니다.

Phase 3 · 관리

컬렉션·태그·리뷰·플레이 기록

필터/정렬, 태그 생성과 관리, 개인 평점·리뷰 작성, 소장 상태 관리 기능을 완성합니다.

Phase 4 · 추천·공유

인원·연령 추천과 공개 페이지

설명 가능한 추천 결과, 공개 링크, 모바일 점검과 접근성/오류 상태를 마무리합니다.

9. 리뷰 시 확인할 결정 사항

결정 완료개인 서비스로 만듭니다. 등록·수정은 단일 관리자만 가능하고, 다중 사용자 계정 기능은 범위에서 제외합니다.
결정 완료전체 컬렉션과 모든 리뷰를 읽기 전용 공개 페이지로 공유합니다.
결정 완료게임 자동 등록은 Boardlife의 get_auto_search.php?word= 검색 결과와 /game/{number} 상세 페이지를 이용하는 2단계 흐름으로 구성합니다. BGG는 토큰 발급 후 보조 출처로 추가합니다.
선호 확인대표 게임명은 한국어 우선으로 표시하고, 원문/영문 이름을 보조 정보로 함께 두는 방향을 제안합니다.

10. 승인 후 시작할 작업

  • 지정한 KyusungJung/my-boardgame-review-site 원격 저장소를 현재 작업 폴더에 연결합니다.
  • Phase 1부터 실제 앱 골격과 Ant Design 테마를 구현합니다.
  • Boardlife 검색 API와 게임 상세 페이지를 서버 어댑터로 연결하고, 스플렌더 본판/확장판 선택 흐름을 테스트합니다.
  • BGG 토큰 발급 후에는 Boardlife에 없는 메타데이터를 보완하는 보조 출처로 연결합니다.
  • 각 Phase 완료 시 로컬 동작 확인 후 변경 사항을 공유하고, 최종 승인 범위에서 원격 저장소에 반영합니다.

11. 변경 기능 정리 · 2026. 06. 26.

영역변경 내용확인 기준
게임 상세Boardlife 이동 링크의 외부 favicon 직접 참조를 제거하고, 사이트 내부의 /boardlife-icon.svg 아이콘을 사용합니다.상세 화면에서 링크 버튼 이미지가 깨지지 않고 고정 크기로 표시됩니다.
안정성외부 사이트의 favicon 응답 형식, 핫링크 제한, 브라우저별 ICO 렌더링 차이에 영향을 받지 않도록 로컬 SVG 자산으로 교체했습니다.배포 환경에서도 이미지 요청이 현재 도메인 안에서 처리됩니다.
접근성아이콘은 장식 요소로 처리하고, 실제 링크 의미는 기존 버튼 텍스트와 링크 속성이 담당하도록 유지합니다.스크린 리더에서 중복된 이미지 설명이 읽히지 않습니다.
iOS 북마크Safari 북마크와 홈 화면 추가에서 사용할 apple-touch-icon.png와 웹 앱 manifest 아이콘을 추가했습니다.iPhone Safari에서 북마크/홈 화면 추가 시 Board Shelf 아이콘이 함께 표시됩니다.
헤더·프로필관리자 로그인/로그아웃 진입을 프로필 드롭다운 메뉴로 이동하고, 비로그인 상태의 프로필은 비활성 상태로 표시합니다.헤더 버튼 hover 시 텍스트가 검게 변하지 않고, 프로필 클릭 또는 hover로 관리자 메뉴가 열립니다.
탐색·공유상세 화면의 별도 뒤로가기 버튼은 제거하고, 앱 내부 상세 접근 주소는 /?game=게임ID 형식으로 유지합니다.브라우저 뒤로가기로 홈 화면에 돌아가며, 내부 상세 링크 접근 시 해당 게임 상세가 표시됩니다.
게임 공유 카드게임 상세 공유 주소를 /games/게임ID 형식으로 제공하고, 공유 페이지와 Open Graph 이미지를 게임 포스터 중심으로 생성합니다.카카오톡 등 외부 공유 미리보기에서 기본 사이트 이미지가 아니라 해당 게임 포스터와 게임명이 노출됩니다.
상세 데이터 유지상세 화면에서 불러온 게임 설명을 세션 캐시와 컬렉션 상태에 병합해, 홈으로 돌아갔다 다시 열어도 같은 설명을 재조회하지 않도록 했습니다.동일 게임 상세를 반복 조회해도 이미 가져온 설명은 유지되고, 저장/갱신된 데이터만 컬렉션과 상세 화면에 반영됩니다.
홈 복귀 안정화상세 화면으로 이동해도 홈 패널을 완전히 언마운트하지 않고 보존하며, 브라우저 뒤로가기 시 즉시 스크롤 전환을 적용합니다.홈 → 게임 상세 → 브라우저 뒤로가기 흐름에서 홈 화면 이미지와 리스트가 다시 그려지는 깜빡임을 줄입니다.
